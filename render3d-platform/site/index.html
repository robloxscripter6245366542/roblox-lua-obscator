A backend built to turn meshes into

How a job moves through the system

  1. 1
    Upload

    Client PUTs a mesh (OBJ/FBX/USD) to S3-compatible storage via a presigned URL.

  2. 2
    Queue

    FastAPI writes a job row in PostgreSQL and enqueues it on Redis via Celery (or gRPC for low-latency paths).

  3. 3
    Process

    A GPU worker pod — scheduled by Kubernetes + NVIDIA GPU Operator — runs Open3D/Trimesh decimation or a headless Blender Cycles/Eevee render.

  4. 4
    Export

    Output is Draco-compressed and written back as glTF/GLB (or OpenUSD), ready for Three.js, Babylon.js, or Omniverse.

  5. 5
    Preview

    The web client polls job status and streams the finished asset straight into a WebGPU/WebGL viewport.
